# QX Digital Instructions Demo

This demo shows how QX translates user intent into executable digital instructions.

## Example: Database Setup Intent

### 1. User Intent (QX Format)
```yaml
🜛 DOT:DATABASE-SETUP
  ⟡ DEC:CREATE-DATABASE
    txt: "I will set up a new database for the application"
  
  ◎ OUT:DATABASE-READY
    txt: "Database is created, configured, and ready for application use"
    metrics: { status: "planned", users_created: "2", permissions: "configured" }
  
  ⇌ TXN:SETUP-DB
    txt: "Create database, users, and set permissions"
  
  ⚙️ DIG:DB-COMMANDS [format=SQL]
    txt: "SQL commands to set up the database"
    instructions:
      commands:
        - "CREATE DATABASE myapp_prod;"
        - "CREATE USER 'app_user'@'localhost' IDENTIFIED BY 'secure_password';"
        - "GRANT ALL PRIVILEGES ON myapp_prod.* TO 'app_user'@'localhost';"
        - "CREATE USER 'readonly_user'@'%' IDENTIFIED BY 'readonly_pass';"
        - "GRANT SELECT ON myapp_prod.* TO 'readonly_user'@'%';"
        - "FLUSH PRIVILEGES;"
```

### 2. Digital Instructions Output
```sql
-- Database Setup Commands (Generated by DIG:DB-COMMANDS)
CREATE DATABASE myapp_prod;
CREATE USER 'app_user'@'localhost' IDENTIFIED BY 'secure_password';
GRANT ALL PRIVILEGES ON myapp_prod.* TO 'app_user'@'localhost';
CREATE USER 'readonly_user'@'%' IDENTIFIED BY 'readonly_pass';
GRANT SELECT ON myapp_prod.* TO 'readonly_user'@'%';
FLUSH PRIVILEGES;
```

### 3. Ready to Execute
The DIG element has successfully translated the user's intent ("I want a database ready for my application") into specific, executable SQL commands that achieve the desired outcome.

## Key Benefits

1. **Intent Clarity**: User expresses what they want (DEC), what success looks like (OUT)
2. **Automatic Translation**: DIG converts intent to executable instructions  
3. **Multiple Formats**: CLI, SQL, API calls, JSON configs, scripts
4. **Constraint Compliance**: Instructions respect ETH/POL/CST directives
5. **Portable**: Same intent works across different execution environments

## Translation Formats Supported

- **CLI**: Shell commands and scripts
- **SQL**: Database setup and operations  
- **API**: REST/GraphQL calls with proper formatting
- **JSON**: Configuration files and data structures
- **Python/Node/etc**: Script generation for complex workflows

The DIG element makes QX a complete intent-to-execution system!
